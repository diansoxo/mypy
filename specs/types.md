# Система типов языка

## 1. Общая характеристика

Язык является строго типизированным (strongly typed) и статически типизированным (статическая проверка типов на этапе компиляции)

Система типов основана на следующих принципах:
- номинальная типизация
- отсутствие неявных приведений типов
- value semantics (передача и присваивание по значению)

## 2. Встроенные типы

Язык поддерживает следующие встроенные типы:

| Тип | Размер | Описание |
|-----|--------|----------|
| `int8`, `int16`, `int32`, `int64` | 8, 16, 32, 64 бит | знаковые целые числа |
| `uint8`, `uint16`, `uint32`, `uint64` | 8, 16, 32, 64 бит | беззнаковые целые числа |
| `float32`, `float64` | 32, 64 бит | числа с плавающей точкой |
| `bool` | — | логический тип (`true`, `false`) |
| `string` | — | строковый тип |
| `void` | — | отсутствие значения |
| `char` | 8 бит | символ (ASCII), литерал в одинарных кавычках: 'a' |


## 3. Составные типы

### 3.1 Массивы

Массив — это фиксированная последовательность элементов одного типа

Синтаксис:
[T; N]

где:
- `T` — тип элементов
- `N` — размер массива (целочисленный литерал)

Особенности:
- размер массива является частью типа
- массивы имеют value semantics (копируются при присваивании и передаче)
- индексация начинается с 0

Пример:
[int32; 10]


### 3.2 Структуры
Структура- именованный составной тип с полями

struct Point {
    x: int32;
    y: int32;
}

Особенности:
- доступ к полям осуществляется через `.`
- структуры копируются при присваивании


### 3.3 Перечисления (enum)
Перечисление задаёт набор фиксированных вариантов

Синтаксис:
enum Name {
    Variant1;
    Variant2;
}

Особенности:
- варианты не содержат данных
- сопоставление выполняется только по имени варианта
- обращение к варианту через точку: Option.Some

Пример:
enum Option {
    Some;
    None;
}

### 3.4 Кортежи
Кортеж — фиксированная последовательность значений

Синтаксис:
(T1, T2, ..., Tn)

Особенности:
- кортеж должен содержать минимум два элемента
- доступ к элементам не поддерживается
- распаковка не поддерживается
- используется только как значение или аргумент функции


## 4. Синонимы типов

Синоним типа объявляется следующим образом:
type Name = ExistingType

Синоним не создаёт новый тип и полностью эквивалентен исходному

Пример:
type Meters = int32

Синонимы допускаются для любых известных типов: базовых, массивов, кортежей и пользовательских
type Meters = int32
type Grid = [int32; 10]

## 5. Типизация

Язык использует номинальную типизацию.

Два типа считаются совместимыми только если их имена совпадают

Пример:
struct A {
    x: int32;
}

struct B{
    x: int32
}
Где А и В разные типы

## 6. Семантика значений

Язык использует value semantics:
- все значения передаются по значению
- присваивание создаёт копию значения
- массивы, структуры и кортежи копируются полностью


## 7. Приведение типов

Приведение типов выполняется явно с помощью оператора `as`

Пример:
x as float64

Разрешены:
- приведения между целыми типами
- приведения между вещественными типами
- приведения между целыми и вещественными типами

Ограничения:
- неявные приведения запрещены
- Поддерживаемые приведения:
int → float: x as float64
float → int: x as int32 (дробная часть отбрасывается)
char → int: c as int32 (числовой код символа)
int → char: n as char (символ по коду)
- пользовательские типы (struct, enum, tuple) не поддерживаются

Поведение:
- при приведении `float → int` дробная часть отбрасывается

## 8. Арифметические операции

Поддерживаются:
- `+`, `-`, `*`, `/`, `%`

Правила:
- операнды должны иметь одинаковый тип
- результат имеет тот же тип

Примеры:
int32 + int32 → int32
float64 * float64 → float64
Запрещено:
int32 + float64

## 9. Логические операции

Поддерживаются:
- `and`, `or`, `not`

Применяются только к типу `bool`

## 10. Операции сравнения

Поддерживаются:
- `==`, `!=`, `<`, `>`, `<=`, `>=`

Правила:
- операнды должны иметь одинаковый тип
- результат имеет тип `bool`

## 11. Строки

Тип `string` поддерживает:
- конкатенацию (`+`)
- сравнение (`==`, `!=`)

Особенности:
- строки являются неизменяемыми
- строки передаются по значению

## 12. Тип void

Тип `void` обозначает отсутствие значения

Особенности:
- используется как возвращаемый тип функций
- не может использоваться в выражениях
- переменные типа `void` запрещены

## 13 Тип char

Тип `char` представляет один символ в кодировке ASCII
Размер: 8 бит, хранится как uint8
Литерал: 'a', '\n', '\t'
Операции: сравнение (==, !=, <, >, <=, >=)
Приведение: char as int32 — даёт числовой код символа, int32 as char — обратное преобразование
Строки и char несовместимы без явного приведения


## 14. Ошибки времени выполнения

Обрабатываются следующие ошибки:

Деление на ноль:
runtime error: division by zero at line

Выход за границы массива:
runtime error: index out of bounds at line

При возникновении ошибки выполнение программы прекращается.


## 15. Совместимость типов

Типы считаются совместимыми только если:
- их имена совпадают, или
- используется явное приведение (`as`)

Неявная совместимость отсутствует

## 16. Числовые литералы

Целочисленные литералы имеют тип int32
Вещественные литералы имеют тип float64
inf и NaN являются литералами типа float64
Шестнадцатеричные (0xFF) и двоичные (0b1010) литералы имеют тип int32

## 17. Дополнения к строкам

Строки поддерживают получение длины:
len(s)

Строки представлены в кодировке UTF-8.

## 18. Формат ошибок времени выполнения

Ошибки имеют формат:
runtime error: <message> at line <N>

## 19. Представление числовых типов в интерпретаторе

Все целочисленные типы (int8, int16, int32, int64, uint8, uint16, uint32, uint64)
хранятся внутри как int64_t. Переполнение не проверяется

Все вещественные типы (float32, float64) хранятся как double

## 20. Представление enum в интерпретаторе

Варианты перечисления хранятся как строки- имя варианта
Например, Option.Some хранится как строковое значение "Some"
Сравнение вариантов выполняется как сравнение строк
