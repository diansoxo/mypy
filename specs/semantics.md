# Семантика языка

## 1. Общая модель вычислений

Язык использует стратегию вычисления **call-by-value**:
- аргументы функций вычисляются перед вызовом
- в функцию передаются копии значений

например:
func add(a: int32, b: int32) -> int32{
    return a + b
}
let x = add(2 + 3, 4)
Сначала вычисляется `2 + 3`, затем вызывается функция

Порядок вычисления:
- операнды вычисляются слева направо
- логические операции `and` и `or` используют сокращённое вычисление (short-circuit)

Например:
false and (1 / 0) тут правая часть не вычисляется


## 2. Области видимости и время жизни

Язык использует **лексическую область видимости (lexical scoping)**

Правила:
- каждый блок создаёт новую область видимости
- переменная существует только внутри блока
- внутренние переменные могут перекрывать внешние (shadowing)
- использование переменной до объявления запрещено

Пример:
let x = 10
if true{
    let x = 5 это перекрытие (shadowing)
}
здесь используется внешний x

## 3. Объявление и инициализация переменных

Переменная должна быть объявлена и инициализирована при объявлении

Пример:
let x: int32 = 10
let y = 20
Ошибка:
let x: int32 - нет инициализации

## 4. Мутабельность
По умолчанию переменные являются неизменяемыми
Для изменения используется `mut`

Пример:
let mut x = 10
let x = 20 - shadowing
Но ошибка:
let x = 10
let x = 20 # ошибка переменная не mut

## 5. Присваивание

Присваивание:
- разрешено только мутабельным переменным
- копирует значение (value semantics)

Пример:

let mut a = 10
let mut b = a
b = 20

Но переменная a остаётся 10

## 6. Функции

Функции:
- принимают аргументы по значению
- имеют собственную область видимости

Правила:
- параметры функции локальны
- тип возвращаемого значения должен совпадать
- функция должна возвращать значение, если указан тип

Пример:
func square(x: int32) -> int32{
    return x * x
}
Ошибка:
func f() -> int32{
    pass   ошибка — нет return
}
## 7. Управление потоком
### 7.1 Условные конструкции

if condition{
...
}else{
...
}
Упрощено:
- elif отсутствует
- используется вложенный if

### 7.2 Цикл while
while condition{
...
}
- условие должно быть `bool`
- выполняется пока условие истинно

### 7.3 Цикл for
for x in array{
...
}
- итерация происходит по массиву
- переменная принимает значения элементов

### 7.4 break и continue
break
continue

- `break` завершает цикл
- `continue` переходит к следующей итерации

Ошибка:
break вне цикла — ошибка компиляции

### 7.5 return
return expression

- завершает выполнение функции
- возвращает значение

## 8. Выражения
Каждое выражение:
- имеет тип
- вычисляется в значение

Проверки:
- типы операндов должны совпадать
- несовместимые операции — ошибка компиляции

Пример: x = 5 + 3
Ошибка:
x = 5 + 3.2 - разные типы

## 9. Доступ к данным

### 9.1 Массивы
arr[i]

Правила:
- индексация начинается с 0
- индекс должен быть целым

Ошибка:
runtime error: index out of bounds at line

### 9.2 Структуры
p.x
Ошибка: p.z - поле не существует

## 10. Сопоставление с образцом (match)

match value {
    case ...
}

Правила:
- проверка выполняется сверху вниз
- выполняется первый подходящий вариант
- `_` — универсальный шаблон

Пример:

match x {
    case 1 -> print("one")
    case _ -> print("other")
}

## 11. Побочные эффекты
Функции могут иметь побочные эффекты:
- изменение переменных
- ввод/вывод

Пример:
print("Hello")

## 12. Встроенные функции
### Ввод/вывод
print(value)
input()

### Управление выполнением
exit(code)
panic(message)

## 13. Ошибки времени выполнения
При ошибке выполнение программы прекращается

Формат:
runtime error: <message> at line

Обязательные ошибки:
Деление на ноль:
runtime error: division by zero at line

Выход за границы массива:
runtime error: index out of bounds at line

## 14. Проверки компиляции
Компилятор обязан проверять:

- переменная объявлена
- переменная инициализирована
- типы корректны
- нет повторных объявлений
- корректность return
- break/continue используются внутри цикла

## 15. Порядок вычисления
- выражения вычисляются слева направо
- `and` и `or` используют short-circuit

Пример:
true or (1 / 0)  - правая часть не вычисляется

## 16. Точка входа программы

Каждая программа должна содержать функцию `main`
Требования:

-функция не принимает аргументов
-имеет сигнатуру `func main() -> int`
-возвращает код завершения программы
Отсутствие функции `main` или неверная сигнатура- ошибка компиляции

## 17. Диагностика ошибок компиляции

Все ошибки компиляции (лексические, синтаксические, семантические)
выводятся в формате:
<файл>:<строка>:<столбец>: error: <сообщение>

При обнаружении ошибки компиляция прекращается

## 18. Выполнение программы

После успешной компиляции выполняется функция `main`
Значение, возвращаемое `main`, передаётся как код завершения программы
