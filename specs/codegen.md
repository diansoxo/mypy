# Исполнение кода

## Выбор подхода
Для финальной фазы компилятора выбрала интерпретацию- прямое обходное исполнение AST

## Архитектура интерпретатора
### Модуль
Реализован в файле `src/interpreter.cppm`, экспортируется как модуль `interpreter` в пространстве имён `interpreter::`.

### Представление значений — `struct Value`
Все значения времени выполнения представлены типом `Value`, содержащим
`std::variant` из следующих альтернатив:

| Вариант | Тип C++ | Значения языка |
|---|---|---|
| `std::monostate` | — | `void` |
| `int64_t` | 64-битное знаковое целое | `int8`, `int16`, `int32`, `int64`, `uint8`…`uint64` |
| `double` | IEEE 754 double | `float32`, `float64`, `inf`, `NaN` |
| `bool` | булево | `bool` |
| `char` | 8-битный символ ASCII | `char` |
| `std::string` | строка | `string` |
| `std::vector<Value>` | вектор значений | массивы, кортежи |
| `std::unordered_map<std::string,Value>` | словарь | `struct` |

Все целые типы языка хранятся единообразно как `int64_t`. `float32` и `float64` хранятся как `double`.

### Управление областями видимости
Стек словарей `std::vector<std::unordered_map<std::string, Value>>`:
scopes_[0] - параметры функции
scopes_[1] - тело функции
scopes_[2] - вложенный блок (if, while, for)

При входе в блок `{` вызывается `pushScope()`, при выходе `}` — `popScope()`.
Поиск переменной идёт сверху вниз - реализует лексическую область видимости со shadowing

### Таблица функций
При запуске `run()` выполняется предварительный проход по `program_.decls`: все `FuncDef`- узлы регистрируются в словаре `std::unordered_map<std::string, const parser::FuncDef*> functions_`.
Функции из `namespace` -блоков регистрируются с составным именем `"Namespace.func"`

Функции из impl-блоков регистрируются под своим именем без префикса
Функции из namespace регистрируются с составным именем "Namespace.func"

### Сигналы управления потоком
`return`, `break`, `continue` реализованы через тип-суммы:

```cpp
struct ReturnSignal { Value value; };
struct BreakSignal  {};
struct ContinueSignal {};
using Signal = std::variant<ReturnSignal, BreakSignal, ContinueSignal>;
```

Каждая функция исполнения инструкции возвращает `std::optional<Signal>`
`std::nullopt` означает нормальное продолжение

### Обработка ошибок времени выполнения
Функция `runtimeError(msg, line)` помечена `[[noreturn]]`:
runtime error: <сообщение> at line <N>
Обязательно обрабатываются:
- Деление на ноль
- Выход за границы массива
- Обращение к необъявленной переменной

## Интерфейс командной строки
mypy <source_file> [--dump-tokens] [--dump-ast]

| Флаг | Поведение |
|---|---|
| *(нет)* | Полный pipeline: лексер → парсер → семантика → интерпретатор |
| `--dump-tokens` | Вывести поток токенов и завершить |
| `--dump-ast` | Вывести AST верхнего уровня и завершить |

Флаг `-o` не применим- программа исполняется напрямую

## Стратегия вычисления

- **Call-by-value**: аргументы передаются по значению
- **Value semantics**: присваивание копирует значение
- Строки иммутабельны; конкатенация создаёт новую строку

## Управление памятью
Вся память управляется автоматически через RAII-механизмы C++:
`std::unique_ptr` для узлов AST, `std::vector`, `std::string`, `std::unordered_map`
Явного выделения и освобождения памяти нет

## Ограничения
- Методы `impl`-блоков поддерживают вызов через точку obj.method(args)
Объект автоматически передаётся первым аргументом
Поиск метода выполняется по имени в таблице функций
- Индексация строк `str[i]` не поддерживается
- Все целые типы хранятся как `int64_t` без проверки переполнения
- Многострочные литералы массивов не поддерживаются

## Обработка enum

Варианты перечисления не регистрируются в таблице функций
При вычислении идентификатора, совпадающего с именем enum, интерпретатор возвращает строковое значение с именем enum
При доступе через точку (Option.Some) возвращается строка "Some"
Сравнение вариантов выполняется как сравнение строк

## Восстановление после ошибок

Парсер реализует стратегию восстановления panic mode:при синтаксической ошибке внутри блока ошибка записывается в список диагностик, затем выполняется синхронизация -пропуск токенов до ближайшего якорного токена (}, func, struct, enum, перенос строки)
Это позволяет накапливать несколько ошибок за один запуск
